# 智能小车全链路系统优化汇总手册 (V3.0 - 最终稳定版)

## 一、 系统全架构通信协议

### 1. 顶层控制 (微信小程序) <-> 业务中转 (Hi3861) [UDP / JSON]
- **下发指令**:
  - `{"carMode": "manual/avoid/line/path"}`: 切换运行模式。
  - `{"carSpeed": "low/middle/high"}`: 切换全局速度档位。
- **状态回传**:
  - `{"L_speed": int, "R_speed": int}`: 物理速度 (mm/s)，小程序端圆形仪表盘实时显示。
  - `{"carMode": "..."}`: **双向同步**，用于反馈车上物理按键触发的模式变化。

### 2. 业务中转 (Hi3861) <-> 运动底座 (STM32 电机板) [UART1 / JSON]
- **极简 JSON 协议**: 缩短键名（`dist`, `volt`, `L_spd`, `R_spd`）以降低 115200 波特率在高频传输下的乱码率。
- **高频率自愈解析**: Hi3861 采用**大括号计数器 (Brace Counter)** 配合 **50ms 超时强制重置**机制。一旦收到的字符序列不合法，解析器会在 50ms 内自动复位，确保不影响后续正常数据的接收。

### 3. 运动底座 (STM32 电机板) <-> 感知终端 (STM32 传感器板) [UART2 / Binary]
- **0xAA (上行)**: 传感器板发送巡线状态及按键模式请求。
- **0xBB (下行)**: 底座同步实时速度（用于转向灯逻辑）及全局模式。

## 二、 核心功能与性能优化汇总

### 1. 控制交互：微信绝对优先与锁定机制
- **指令锁定窗**: 当底座收到来自微信的模式指令时，开启 **500ms 忽略期**。在此期间完全屏蔽物理按键的干扰，确保底座有足够时间通过 `0xBB` 包将新模式强制同步给传感器板计数器。
- **边沿触发**: 物理按键仅在“状态跳变”的瞬间生效一次。这彻底解决了小程序切换模式后又被按键旧值“拉回”的震荡现象。

### 2. 速度系统：全线翻倍与统锁控制
- **全模式联动**: 巡线、避障、路径规划的速度不再硬编码。所有自动模式的速度基准均受小程序“低/中/高”档位控制。
- **基准值翻倍**: 
  - **低速**: 500 mm/s | **中速**: 800 mm/s | **高速**: 1100 mm/s。
- **PID 平滑闭环**: 转向动作已全部由固定 PWM 改为 **PID 闭环控制**，转向动力感与前进完全对称平滑。

### 3. 通信加固：硬件级自动恢复 (Self-Healing)
- **错误拦截回调**: 电机驱动板增加了 `HAL_UART_ErrorCallback`。一旦发生由于电磁干扰或波特率误差导致的串口溢出（ORE）、噪声（NE）或帧错误（FE），底座会自动清除标志并**秒级重启接收**，确保永不掉线。
- **纯净链路**: 彻底禁用了底座的所有调试日志打印，确保 UART1 仅承载纯净的 JSON 状态流。

### 4. 倒车逻辑优化：方向与灯光同步
- **镜像修正**: 修正了摇杆在下半区（倒车）时的左右转向逻辑，使其符合“后退时指向哪，车尾往哪摆”的直觉。
- **灯光同步**: 移除了传感器板冗余的倒车灯信号取反逻辑，使转向灯始终与车辆实际旋转方向（及摇杆指向）保持一致。

## 三、 关键功能参数清单
- **起步最低 PWM**: 150 (15%)，实现无冲击起步。
- **路径规划**: `PULSES_PER_90DEG = 920`，实现精准原地 90 度转向，无限循环绘制正方形。
- **物理消抖**: 传感器板 USR 键采用 200ms 硬件定时器级消抖。
- **电压同步**: 修复了 Hi3861 接收缓冲区满导致的电压读取中断问题。
