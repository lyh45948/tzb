# 智慧工厂安全监测控制系统 - 接口说明文档

## 概述

本文档定义智慧工厂安全监测控制系统中各层级之间的通信接口，包含微信小程序、Python后端、Hi3861网关、STM32电机驱动板和STM32传感器板之间的完整通信协议。

---

## 一、系统架构

```
┌─────────────────────────────────────────────────────────────────┐
│                        客户端层                                  │
│  微信小程序 (factory_dashboard/control/monitor/equipment_control) │
│                         TCP :8888                               │
└─────────────────────────────┬───────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                     Python后台服务 (端口8888)                     │
│  TCPServer │ UDPMiniAppService │ DataService                   │
│  UDPService (连接Hi3861 :7788)                                   │
└─────────────────────────────┬───────────────────────────────────┘
                              │ UDP :7788
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                     Hi3861网关 (OpenHarmony)                     │
│         UDP接收任务 │ UDP发送任务 │ UART通信任务                   │
└─────────────────────────────┬───────────────────────────────────┘
                              │ UART2 JSON :115200
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                     STM32电机驱动板 / AGV                        │
│              PID控制 │ 编码器反馈 │ 模式管理                       │
└─────────────────────────────┬───────────────────────────────────┘
                              │ UART2 Binary :115200
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                     STM32传感器板                                │
│       火焰检测 │ 燃气检测 │ CO2/TVOC │ 超声波测距 │ 红外巡线       │
└─────────────────────────────────────────────────────────────────┘
```

### 接口层次划分

| 层次 | 接口类型 | 通信方式 | 协议格式 |
|------|---------|---------|---------|
| 客户端 ↔ 后台 | 应用接口 | TCP | JSON |
| 后台 ↔ Hi3861 | 设备控制接口 | UDP | JSON |
| Hi3861 ↔ STM32电机板 | 板级接口 | UART | JSON |
| STM32电机板 ↔ 传感器板 | 内部接口 | UART | Binary |

## 二、客户端 ↔ 后台服务接口

**通信方式**: TCP Socket
**端口**: 8888
**消息格式**: JSON，每条消息以换行符 `\n` 结尾
**传输编码**: UTF-8

### 2.1 连接管理接口

#### 2.1.1 心跳检测 (Ping)

**请求**:
```json
{
  "type": "ping"
}
```

**响应**:
```json
{
  "type": "pong",
  "timestamp": 1704067200000
}
```

#### 2.1.2 连接Hi3861设备 (Connect)

**请求**:
```json
{
  "type": "connect",
  "carIp": "192.168.1.100",
  "carPort": 7788,
  "deviceId": "device_001"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| carIp | string | 是 | 设备IP地址 |
| carPort | int | 否 | 端口，默认7788 |
| deviceId | string | 否 | 设备ID，默认自动生成 |

**响应**:
```json
{
  "type": "connect_result",
  "success": true,
  "message": "已连接到设备 192.168.1.100:7788",
  "deviceId": "device_001"
}
```

#### 2.1.3 断开连接 (Disconnect)

**请求**:
```json
{
  "type": "disconnect"
}
```

**响应**:
```json
{
  "type": "disconnect_result",
  "success": true,
  "message": "已断开与设备的连接"
}
```

### 2.2 设备控制接口

#### 2.2.1 控制命令下发 (Control)

**请求**:
```json
{
  "type": "control",
  "command": {
    "carStatus": "run"
  }
}
```

**command 支持的命令格式**:

| 命令类型 | 命令格式 | 说明 |
|---------|---------|------|
| 开机 | `{"carStatus": "on"}` | 启动设备 |
| 关机 | `{"carStatus": "off"}` | 关闭设备 |
| 前进 | `{"carStatus": "run"}` | 向前移动 |
| 后退 | `{"carStatus": "back"}` | 向后移动 |
| 左转 | `{"carStatus": "left"}` | 向左转向 |
| 右转 | `{"carStatus": "right"}` | 向右转向 |
| 停止 | `{"carStatus": "stop"}` | 立即停止 |
| 低速档 | `{"carSpeed": "low"}` | 速度约500mm/s |
| 中速档 | `{"carSpeed": "middle"}` | 速度约800mm/s |
| 高速档 | `{"carSpeed": "high"}` | 速度约1100mm/s |
| 手动模式 | `{"carMode": "manual"}` | 手动控制模式 |
| 避障模式 | `{"carMode": "avoid"}` | 超声波避障模式 |
| 巡线模式 | `{"carMode": "line"}` | 红外巡线模式 |
| 路径模式 | `{"carMode": "path"}` | 路径规划模式 |
| 摇杆控制 | `{"joyX": 50, "joyY": 100}` | 差速控制 |
| 风扇控制 | `{"fan": 1}` | 风扇开关 (0=关, 1=开) |
| LED控制 | `{"led": 1}` | LED开关 (0=关, 1=开) |
| 蜂鸣器控制 | `{"buzzer": 1}` | 蜂鸣器开关 (0=关, 1=开) |
| 智能照明 | `{"smartLight": {"mode": "auto", "brightness": 75}}` | LED亮度控制 |

**摇杆控制参数说明**:

| 字段 | 类型 | 范围 | 说明 |
|------|------|------|------|
| joyX | int | -100 ~ 100 | X轴方向，负值左转，正值右转 |
| joyY | int | -100 ~ 100 | Y轴方向，负值后退，正值前进 |

**响应**:
```json
{
  "type": "control_result",
  "success": true,
  "message": "命令已发送"
}
```

## 五、STM32电机板 ↔ STM32传感器板接口

**通信方式**: UART2
**波特率**: 115200
**数据格式**: Binary (二进制)

### 5.1 同步包（电机板 → 传感器板）

**帧长度**: 8字节

| 字节索引 | 内容 | 说明 |
|----------|------|------|
| 0 | 0xBB | 帧头 |
| 1-2 | 左轮速度 | 有符号16位整数 |
| 3-4 | 右轮速度 | 有符号16位整数 |
| 5 | 小车状态 | 0=停止, 1=前进, 2=后退, 3=左转, 4=右转 |
| 6 | 运行模式 | 0x00=远程, 0xD0=避障, 0xF0=巡线, 0xE0=路径 |
| 7 | 校验和 | 前7字节异或校验 |

### 5.2 避障数据包（传感器板 → 电机板）

**帧长度**: 4字节

| 字节索引 | 内容 | 说明 |
|----------|------|------|
| 0 | 0xFF | 帧头 |
| 1-2 | 距离值 | 无符号16位整数 (mm) |
| 3 | 校验和 | 前3字节异或校验 |

### 5.3 巡线数据包（传感器板 → 电机板）

**帧长度**: 4字节

| 字节索引 | 内容 | 说明 |
|----------|------|------|
| 0 | 0xAA | 帧头 |
| 1 | 巡线结果 | 7个光电管状态，bit0-6对应各通道 |
| 2 | 模式字节 | 0x00=远程, 0xD0=避障, 0xF0=巡线, 0xE0=路径 |
| 3 | 校验和 | 前3字节异或校验 |

## 六、语音控制接口

**接口**: UART1 (GPIO 5: RX, GPIO 6: TX)
**波特率**: 115200
**数据位**: 8 bit
**停止位**: 1 bit
**校验位**: None

### 6.1 指令集定义 (单字节)

| 指令 (HEX) | 功能描述 | 动作说明 |
| :--- | :--- | :--- |
| **0x10** | **开启电机电源** | 底盘上电，准备接收运动指令 |
| **0x11** | **关闭电机电源** | 底盘断电，切断所有动力 |
| **0x20** | **前进** | 切换到手动模式并执行前进 |
| **0x21** | **后退** | 切换到手动模式并执行后退 |
| **0x22** | **左转** | 切换到手动模式并执行左转 |
| **0x23** | **右转** | 切换到手动模式并执行右转 |
| **0x24** | **停止** | **最高优先级**：立即切换为手动模式，强行打断避障、巡线或路径规划并停车 |
| **0x30** | **进入避障模式** | 开启避障算法，自动规避前方障碍 |
| **0x31** | **进入巡线模式** | 开启巡线算法，沿黑线行驶 |
| **0x40** | **走正方形** | 发送一组预设的路径点，让设备跑出正方形 |
| **0x41** | **走三角形** | 发送一组预设的路径点，让设备跑出三角形 |
| **0x50** | **打开风扇** | 开启外设风扇 |
| **0x51** | **关闭风扇** | 关闭外设风扇 |
| **0x60** | **查询温湿度** | 获取环境温湿度并回传给语音模块 |

### 6.2 回传协议 (Hi3861 -> 语音模块)

当执行 **0x60** (查询) 时，Hi3861 会发送两个独立的 6 字节数据包：

#### 温度包 (Message ID: 0x01)
*   格式：`AA 55 01 [TempValue] 55 AA`
*   `[TempValue]`：温度整数值 (1 字节)

#### 湿度包 (Message ID: 0x02)
*   格式：`AA 55 02 [HumiValue] 55 AA`
*   `[HumiValue]`：湿度整数值 (1 字节)
