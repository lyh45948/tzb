## 2. 数据模拟算法

### 2.1 概述

数据模拟服务（`SimulationService`）用于在演示模式下生成模拟传感器数据，以及在非演示模式下基于实测数据推算 CO2 浓度。核心目标是：

- **温度模拟**：反映日间温度变化（6点最低，18点最高）
- **湿度模拟**：与温度呈反相变化
- **光照模拟**：模拟日出日落周期
- **CO2 模拟**：基于多因素推算，并使用滑动窗口平滑

### 2.2 温度模拟算法

```python
def _simulate_temperature(self, hour):
    base_temp = 22  # 基础温度 22℃
    # 正弦波模拟日变化：6点最低，18点最高
    time_factor = 8 * math.sin((hour - 6) * math.pi / 12)
    noise = random.uniform(-2, 2)  # 随机噪声 ±2℃
    temperature = base_temp + time_factor + noise
    return round(max(15, min(35, temperature)), 1)
```

**算法说明**：
- **基础值**：22℃
- **振幅**：±8℃（6点约14℃，18点约30℃）
- **范围约束**：15℃ ~ 35℃
- **曲线**：正弦波，周期24小时

```

```

### 2.3 湿度模拟算法

```python
def _simulate_humidity(self, hour, temperature):
    base_humidity = 60  # 基础湿度 60%
    # 与温度反相的正弦波
    time_factor = -15 * math.sin((hour - 6) * math.pi / 12)
    # 温度影响：温度 > 25℃ 时，每升 1℃ 降低 0.5%
    temp_factor = -0.5 * max(0, temperature - 25)
    noise = random.uniform(-3, 3)  # 随机噪声 ±3%
    humidity = base_humidity + time_factor + temp_factor + noise
    return round(max(30, min(90, humidity)), 1)
```

**算法说明**：
- **基础值**：60%
- **时间因子**：与温度反相（温度高时湿度低）
- **温度补偿**：高温加速蒸发
- **范围约束**：30% ~ 90%

### 2.4 光照模拟算法

```python
def _simulate_light(self, hour):
    sunrise, sunset = 6.0, 18.5

    if hour < sunrise or hour > sunset:
        # 夜间微弱光照（演示用）
        if 23 <= hour or hour < 5:
            return int(random.uniform(80, 120))   # 深夜：80-120 lux
        else:
            return int(random.uniform(100, 200))  # 黎明/黄昏：100-200 lux

    # 白天光照计算
    day_progress = (hour - sunrise) / (sunset - sunrise)
    light_factor = math.sin(day_progress * math.pi)  # 正弦曲线
    cloud_factor = random.uniform(0.6, 1.0)  # 天气因子（云层遮挡）
    lux = 800 + (1200 - 800) * light_factor * cloud_factor
    return int(max(100, min(1500, lux)))
```

**算法说明**：
- **日出/日落**：6:00 / 18:30
- **夜间光照**：80-200 lux（演示模式避免完全黑暗）
- **白天基础**：800 lux
- **峰值光照**：1200 lux（正午）
- **天气因子**：0.6 ~ 1.0（模拟云层遮挡）
- **范围约束**：50 ~ 1500 lux

### 2.5 CO2 模拟与平滑算法

```python
class DataSmoother:
    """滑动窗口平均平滑器"""
    def __init__(self, window_size=10):
        self.window_size = window_size
        self.data_buffers = {}  # {key: deque(maxlen=window_size)}

    def add_and_get_average(self, key, value):
        if key not in self.data_buffers:
            self.data_buffers[key] = deque(maxlen=self.window_size)
        self.data_buffers[key].append(value)
        avg = sum(self.data_buffers[key]) / len(self.data_buffers[key])
        return int(round(avg)) if isinstance(value, int) else round(avg, 1)
```

```python
def _simulate_co2(self, hour, temperature, humidity, lux):
    base_co2 = 450
    # 时间因子
    time_factor = -30 if 6 <= hour <= 18 else 80   # 白天/夜晚
    temp_factor = 1 * max(0, temperature - 22)       # 温度影响
    humi_factor = -0.3 * max(0, humidity - 60)       # 湿度影响
    light_factor = -0.01 * lux                       # 光照影响
    noise = random.uniform(-50, 50)                  # 随机噪声
    co2 = base_co2 + time_factor + temp_factor + humi_factor + light_factor + noise
    return int(max(400, min(1000, co2)))
```

**算法说明**：
- **基础值**：450 ppm
- **时间因子**：白天 -30 ppm，夜晚 +80 ppm
- **温度因子**：每高于 22℃ 1℃，增加 1 ppm
- **湿度因子**：每高于 60% 1%，减少 0.3 ppm
- **光照因子**：每 lux 减少 0.01 ppm
- **平滑处理**：10 次采样滑动窗口平均
- **范围约束**：400 ~ 1000 ppm

## 3. 智能灌溉算法

### 3.1 概述

智能灌溉服务（`IrrigationService`）根据环境参数和历史记录自动计算推荐灌溉水量。算法综合考虑温度、湿度、光照、灌溉历史和季节因素。

### 3.2 核心公式

```
推荐水量 = 基础水量 × 温度因子 × 湿度因子 × 光照因子 × 历史因子 × 季节因子
```

### 3.3 温度因子

```python
def _calc_temperature_factor(self, temperature: float) -> float:
    # 25°C 为基准，每高 1°C 增加 2%
    return 1 + (temperature - 25) * 0.02
```

| 温度 | 因子 |
|------|------|
| 20℃ | 0.9 |
| 25℃ | 1.0 |
| 30℃ | 1.1 |
| 35℃ | 1.2 |

**原理**：温度越高，蒸发越快，需水量越多。

### 3.4 湿度因子

```python
def _calc_humidity_factor(self, humidity: float) -> float:
    # 60% 为基准，每低 1% 增加 1.5%
    return 1 + (60 - humidity) * 0.015
```

| 湿度 | 因子 |
|------|------|
| 30% | 1.45 |
| 45% | 1.225 |
| 60% | 1.0 |
| 75% | 0.925 |

**原理**：湿度越低（空气越干燥），植物蒸腾越快，需水越多。

### 3.5 光照因子

```python
def _calc_light_factor(self, light: float) -> float:
    # 光照越强需水越多
    return 1 + (light / 10000) * 0.1
```

| 光照 (lux) | 因子 |
|-----------|------|
| 0 | 1.0 |
| 5000 | 1.05 |
| 10000 | 1.1 |
| 20000 | 1.2 |

**原理**：强光促进光合作用和蒸腾作用，需水量增加。

### 3.6 历史因子

```python
def _calc_history_factor(self, device_id: str) -> float:
    recent = self.get_recent_irrigations(device_id, limit=3)
    if not recent:
        return 1.0

    now = datetime.utcnow()
    # 最近 1 小时内灌溉过 → 降低水量 50%
    if (now - recent[0].start_time) < timedelta(hours=1):
        return 0.5

    # 3 小时内灌溉总量超过 2000ml → 降低水量 30%
    total_water = sum(i.water_amount for i in recent if i.water_amount)
    if total_water > 2000:
        return 0.7

    return 1.0
```

| 历史条件 | 因子 |
|---------|------|
| 无历史记录 | 1.0 |
| 1小时内灌溉过 | 0.5 |
| 3小时内 > 2000ml | 0.7 |
| 其他正常情况 | 1.0 |

**原理**：避免过度灌溉，保护植物根系。

### 3.7 季节因子

```python
SEASON_FACTORS = {
    'spring': {'factor': 1.1, 'name': '春季'},   # 万物生长，需水增加 10%
    'summer': {'factor': 1.3, 'name': '夏季'},   # 高温蒸发，需水增加 30%
    'autumn': {'factor': 1.0, 'name': '秋季'},   # 基准值
    'winter': {'factor': 0.7, 'name': '冬季'}    # 低温休眠，需水减少 30%
}
```

| 季节 | 因子 | 说明 |
|------|------|------|
| 春 (3-5月) | 1.1 | 万物复苏，需水增加 |
| 夏 (6-8月) | 1.3 | 高温蒸腾，需水大增 |
| 秋 (9-11月) | 1.0 | 基准值 |
| 冬 (12-2月) | 0.7 | 低温休眠，需水减少 |

### 3.8 计算示例

假设：春末（季节=1.1）、温度=30℃、湿度=40%、光照=10000 lux、无历史记录

```
基础水量 = 500 ml

温度因子 = 1 + (30-25) × 0.02 = 1.10
湿度因子 = 1 + (60-40) × 0.015 = 1.30
光照因子 = 1 + (10000/10000) × 0.1 = 1.10
历史因子 = 1.0
季节因子 = 1.1

推荐水量 = 500 × 1.10 × 1.30 × 1.10 × 1.0 × 1.1
         = 500 × 1.71605
         ≈ 858 ml
```

**最终范围约束**：100 ml ~ 2000 ml

## 4. 智能光照算法

### 4.1 概述

智能光照服务（`smart_light_task.c`）根据环境光照强度和时间段自动调节 LED 补光亮度。算法核心是**分段映射 + 时间系数 + 平滑过渡**。

### 4.2 时间段定义

```c
typedef enum {
    TIME_PERIOD_DAWN,      // 黎明 (5:00-7:00)
    TIME_PERIOD_MORNING,   // 上午 (7:00-12:00)
    TIME_PERIOD_NOON,      // 中午 (12:00-14:00)
    TIME_PERIOD_AFTERNOON, // 下午 (14:00-17:00)
    TIME_PERIOD_DUSK,      // 黄昏 (17:00-19:00)
    TIME_PERIOD_EVENING,   // 晚间 (19:00-23:00)
    TIME_PERIOD_NIGHT      // 深夜 (23:00-5:00)
} time_period_t;
```

### 4.3 时间系数

```c
static uint8_t get_time_factor(time_period_t period)
{
    switch (period) {
        case TIME_PERIOD_MORNING:
        case TIME_PERIOD_NOON:
        case TIME_PERIOD_AFTERNOON:
            return 100;  // 白天正常补光
        case TIME_PERIOD_DAWN:
        case TIME_PERIOD_DUSK:
            return 70;   // 过渡时段适度补光
        case TIME_PERIOD_EVENING:
            return 40;   // 晚间低强度补光
        case TIME_PERIOD_NIGHT:
        default:
            return 0;    // 深夜不自动补光
    }
}
```

| 时间段 | 系数 | 说明 |
|--------|------|------|
| 上午/中午/下午 | 100% | 正常补光 |
| 黎明/黄昏 | 70% | 过渡时段 |
| 晚间 | 40% | 低强度 |
| 深夜 | 0% | 不自动补光 |

### 4.4 光照等级划分

```c
static light_level_t get_light_level(uint16_t env_lux)
{
    if (env_lux < 50)       return LIGHT_LEVEL_DARK;
    else if (env_lux < 200) return LIGHT_LEVEL_DIM;
    else if (env_lux < 500) return LIGHT_LEVEL_LOW;
    else if (env_lux < 1000) return LIGHT_LEVEL_NORMAL;
    else if (env_lux < 2000) return LIGHT_LEVEL_BRIGHT;
    else return LIGHT_LEVEL_VERY_BRIGHT;
}
```

| 环境光照 (lux) | 等级 |
|---------------|------|
| < 50 | 黑暗 |
| 50-200 | 昏暗 |
| 200-500 | 弱光 |
| 500-1000 | 正常 |
| 1000-2000 | 明亮 |
| > 2000 | 强光 |

### 4.5 核心算法

```c
static uint8_t smart_light_calculate(uint16_t env_lux, time_period_t period)
{
    // 1. 获取时间因素系数
    uint8_t time_factor = get_time_factor(period);

    // 深夜不自动补光
    if (time_factor == 0) {
        return 0;
    }

    // 2. 根据环境光照计算基础补光需求
    uint8_t base_brightness = 0;

    // 分段映射：环境光照越低，补光需求越高
    if (env_lux < 50)       base_brightness = 100;  // 黑暗：全亮度
    else if (env_lux < 100) base_brightness = 90;   // 极暗：90%
    else if (env_lux < 200) base_brightness = 75;   // 昏暗：75%
    else if (env_lux < 400) base_brightness = 50;   // 偏暗：50%
    else if (env_lux < 700) base_brightness = 30;   // 略暗：30%
    else if (env_lux < 1000) base_brightness = 15;  // 接近正常：15%
    else if (env_lux < 1500) base_brightness = 5;   // 正常：5%维持
    else base_brightness = 0;                        // 明亮：关闭

    // 3. 综合时间因素计算最终亮度
    uint8_t final_brightness = (base_brightness * time_factor) / 100;

    return final_brightness;
}
```

**亮度映射表**：

| 环境光照 (lux) | 基础亮度 | 白天 (×100%) | 黎明/黄昏 (×70%) | 晚间 (×40%) |
|---------------|----------|-------------|-----------------|-------------|
| < 50 (黑暗) | 100% | 100% | 70% | 40% |
| 50-100 (极暗) | 90% | 90% | 63% | 36% |
| 100-200 (昏暗) | 75% | 75% | 52% | 30% |
| 200-400 (偏暗) | 50% | 50% | 35% | 20% |
| 400-700 (略暗) | 30% | 30% | 21% | 12% |
| 700-1000 (接近正常) | 15% | 15% | 10% | 6% |
| 1000-1500 (正常) | 5% | 5% | 3% | 2% |
| > 1500 (明亮) | 0% | 0% | 0% | 0% |

### 4.6 平滑过渡算法

```c
static uint8_t smooth_transition(uint8_t current, uint8_t target)
{
    int16_t diff = target - current;

    if (diff > 5) {
        return current + 5;  // 渐增，每次最多增加 5%
    } else if (diff < -5) {
        return current - 5;  // 渐减，每次最多减少 5%
    }
    return target;  // 接近目标，直接设置
}
```

**原理**：避免亮度突变造成视觉不适，以 5% 为步长逐步调整。

## 5. 算法对比总结

| 特性 | 数据模拟 | 智能灌溉 | 智能光照 |
|------|---------|---------|---------|
| **输入参数** | 时间、温度、湿度、光照 | 温度、湿度、光照、历史、季节 | 环境光照、时间段 |
| **核心方法** | 正弦波 + 滑动平均 | 多因子乘法模型 | 分段映射 + 时间系数 |
