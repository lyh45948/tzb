### 3. ✅ 网络断线重连机制

**实现位置**：`utils/network-manager.js`

**核心特性**：

#### 3.1 指数退避重连
```javascript
reconnectDelay = 1000 * Math.pow(2, reconnectAttempts)
```
- 第1次：1秒后
- 第2次：2秒后
- 第3次：4秒后
- 第4次：8秒后
- ...
- 最多重连10次

#### 3.2 自动重连触发条件
- 连接超时（10秒无响应）
- 网络错误
- UDP连接关闭

#### 3.3 重连限制
- 最大重连次数：10次
- 超过限制后停止重连
- 显示"重连失败，请检查网络"提示

#### 3.4 手动重连
- 首页"刷新状态"按钮
- 控制页自动检测连接状态
- 配置修改后自动重连

**关键代码**：
```javascript
startReconnect() {
  if (this.reconnectAttempts >= this.maxReconnectAttempts) {
    console.error('[NetworkManager] 达到最大重连次数');
    this.notifyConnectionChange(false, '重连失败，请检查网络');
    return;
  }

  const delay = this.reconnectDelay * Math.pow(2, this.reconnectAttempts);
  console.log('[NetworkManager] 将在', delay, 'ms后尝试第', 
              this.reconnectAttempts + 1, '次重连');
  
  this.reconnectTimer = setTimeout(() => {
    this.reconnectAttempts++;
    this.connect();
  }, delay);
}
```

### 4. ✅ 心跳保活功能

**实现位置**：`utils/network-manager.js`

**核心机制**：

#### 4.1 心跳参数
```javascript
heartbeatInterval: 3000        // 心跳间隔：3秒
connectionTimeout: 10000     // 连接超时：10秒
lastHeartbeatTime: 0          // 最后心跳发送时间
lastReceiveTime: 0           // 最后数据接收时间
```

#### 4.2 心跳包格式
```json
{
  "cmd": "ping",
  "timestamp": 1735432100000
}
```

#### 4.3 超时检测逻辑
```javascript
checkConnectionTimeout() {
  const now = Date.now();
  const timeSinceLastReceive = now - this.lastReceiveTime;
  
  if (timeSinceLastReceive > this.connectionTimeout) {
    console.warn('[NetworkManager] 连接超时');
    
    if (this.isConnected) {
      this.isConnected = false;
      this.notifyConnectionChange(false, '连接超时');
      this.startReconnect();  // 启动重连
    }
  }
}
```

#### 4.4 心跳循环
```javascript
startHeartbeat() {
  this.heartbeatTimer = setInterval(() => {
    this.sendHeartbeat();           // 发送心跳
    this.checkConnectionTimeout();  // 检查超时
  }, this.heartbeatInterval);
}
```

#### 4.5 保活效果
- **保持活跃**：每3秒发送一次心跳，保持连接不被系统关闭
- **快速发现断线**：10秒内未收到任何数据即判定断线
- **自动恢复**：检测到断线后立即启动重连机制
- **状态实时**：UI显示最后接收时间，让用户了解连接健康度

### 5. ✅ 统一错误处理

**实现位置**：`utils/error-handler.js`

**核心功能**：

#### 5.1 错误类型分类
```javascript
错误类型映射:
- network_error      // 网络错误 (error级别)
- network_timeout   // 网络超时 (warn级别)
- parse_error       // 消息解析错误 (error级别)
- send_error        // 发送错误 (warn级别)
- receive_error     // 接收错误 (warn级别)
- app_error         // 应用错误 (error级别)
- promise_rejection // Promise拒绝 (error级别)
- config_error      // 配置错误 (error级别)
- validation_error  // 验证错误 (warn级别)
- permission_error  // 权限错误 (error级别)
```

#### 5.2 错误处理流程
```
1. 捕获错误 → 2. 记录到历史 → 3. 日志输出 → 4. 触发回调 → 5. 保存到本地
```

#### 5.3 错误记录结构
```javascript
{
  type: 'network_error',        // 错误类型
  message: '连接超时',        // 错误信息
  detail: {...},               // 错误详情
  timestamp: 1735432100000,  // 时间戳
  level: 'error'              // 日志级别
}
```

#### 5.4 日志级别控制
```javascript
logLevel: 'info'  // 可选: debug, info, warn, error
```

#### 5.5 错误历史管理
- 内存中保留最近50条
- 本地存储保留最近100条
- 支持导出JSON格式
- 支持统计分析

#### 5.6 应用级错误监听
```javascript
// 监听小程序全局错误
wx.onError(this.handleAppError.bind(this));

// 监听未处理的Promise拒绝
wx.onUnhandledRejection(this.handlePromiseRejection.bind(this));
```

#### 5.7 错误统计功能
```javascript
getErrorStats() {
  return {
    'network_error': {
      count: 15,
      messages: ['连接超时', '发送失败', ...]
    },
    'parse_error': {
      count: 3,
      messages: ['JSON解析失败', ...]
    }
  }
}
```

#### 5.8 使用示例
```javascript
// 捕获网络错误
try {
  networkManager.send(data);
} catch (error) {
  errorHandler.handle('network_error', '网络错误: ' + error.message);
}

// 监听错误通知
errorHandler.onError((errorInfo) => {
  console.log('收到错误:', errorInfo);
  
  if (errorInfo.type.startsWith('network')) {
    wx.showToast({
      title: errorInfo.message,
      icon: 'none'
    });
  }
});
```

## 数据流程

### 连接流程
```
用户在首页输入IP → 点击"连接设备" 
  → networkManager.connect()
    → 创建UDP Socket
    → 绑定本地端口
    → 监听消息/错误/关闭事件
    → 发送心跳测试
    → 启动心跳定时器
    → 连接成功/失败
    → 通知UI更新
```

### 断线重连流程
```
心跳超时检测 (10秒无数据)
  → 标记连接断开
    → 停止心跳
      → 通知UI显示断线
        → 启动指数退避重连
          → 延迟N秒后重连
            → connect()
              → 成功: 重置重连计数
              → 失败: 增加重连计数，继续重连
                → 达到10次: 停止重连，提示用户
```

### 消息发送流程
```
用户操作 (摇杆/按钮)
  → 检查连接状态
    → 已连接: 通过networkManager.send()
      → UDP发送
      → 成功: 返回true
      → 失败: errorHandler.handle()
    → 未连接: 提示"未连接"
```
