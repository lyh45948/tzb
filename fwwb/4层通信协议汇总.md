# 智能小车四层通信协议汇总手册 (V2.0 - 2025/12/31)

本文档定义了智能小车系统（微信小程序、Hi3861 逻辑板、STM32 电机驱动板、STM32 传感器扩展板）各层级之间的通信标准。

## 一、 系统架构

1.  **应用层 (微信小程序)**：用户终端，通过 WiFi/UDP 下发高层业务指令并展现传感器数据。
2.  **网关层 (Hi3861)**：协议中转站，负责 UDP 与 UART 协议的双向转换及语音指令映射。
3.  **执行层 (STM32 电机板)**：运动核心，负责 PID 闭环控制、路径规划算法及电源管理。
4.  **感知层 (STM32 传感器板)**：硬件终端，负责超声波避障、红外巡线采集及氛围灯效。

---

## 二、 通信协议细节

### 1. 应用层 (小程序) ↔ 网关层 (Hi3861)
*   **通信媒介**：WiFi (UDP 单播/广播), 端口 7788
*   **数据格式**：标准 JSON
*   **下发 (Downlink)**：
    *   电源：`{"carStatus": "on/off"}`
    *   运动：`{"carStatus": "run/back/left/right/stop"}`
    *   模式：`{"carMode": "manual/avoid/line/path"}`
    *   摇杆：`{"joyX": int, "joyY": int}` (-100 ~ 100)
    *   档位：`{"carSpeed": "low/middle/high"}`
    *   路径：`{"path": [{"d": int, "a": int}, ...]}`
*   **上报 (Uplink)**：
    *   状态包：`{"carPower": int, "L_spd": int, "R_spd": int, "carMode": "string", "distance": int}` (注：速度为带符号整数，正数前进，负数后退)
    *   环境包：`{"temp": int, "humi": int, "lux": int, "env": {...}}` (详见 `udp_send_task.c`)

### 2. 网关层 (Hi3861) ↔ 执行层 (STM32 电机板)
*   **通信媒介**：UART (115200, N, 8, 1)
*   **数据格式**：紧凑型 JSON (省略空格提升带宽利用率)
*   **控制指令 (`control` 对象)**：
    *   转向：`{"control":{"turn":"run/back/left/right/stop"}}`
    *   执行电源：`{"control":{"power":"on/off"}}`
    *   模式：`{"control":{"mode":"manual/avoid/line/path"}}`
    *   速度档位映射：`{"control":{"pwm":{"L_Motor": 390/420/450, "R_Motor": 390/420/450}}}`
    *   自动模式速度：`{"control":{"speed": int}}` (单位 mm/s)
    *   摇杆同步：`{"control":{"joyX": int, "joyY": int}}`
    *   路径分步：`{"control":{"d": int, "a": int}}`
*   **反馈数据 (`status` 对象)**：
    *   周期性上报 (100ms)：`{"status":{"dist":%d,"volt":%d,"L_spd":%d,"R_spd":%d,"mode":"%s"}}` (注：速度为带符号整数)

### 3. 执行层 (STM32 电机板) ↔ 感知层 (STM32 传感器板)
*   **通信媒介**：UART (双向异步)
*   **执行层下发 (Sync Packet)**：
    *   **格式**：`0xBB` (头) + `L_spd(2B)` + `R_spd(2B)` + `Status(1B)` + `Mode(1B)` + `Sum(1B)`
    *   **用途**：同步速度用于变频灯效，同步状态用于转向灯/倒车灯逻辑。
*   **感知层上报 (Sensor Packets)**：
    *   **避障包**：`0xFF` (头) + `Dist_H(1B)` + `Dist_L(1B)` + `Sum(1B)`
    *   **巡线/按键包**：`0xAA` (头) + `LineOut(1B)` + `ModeByte(1B)` + `Sum(1B)`
    *   **模式编码 (ModeByte)**：`0x00`-遥控, `0xD0`-避障, `0xF0`-巡线, `0xE0`-路径 (与感知层 `gLineType` 或运算组合)。

### 4. 语音控制扩展 (External → Hi3861)
*   **接口**：UART1 (115200)
*   **协议**：单字节 Hex 指令集 (参考 `语音接口说明.md`)
*   **自动翻译**：网关层收到识别码 (如 `0x24`) 后，翻译为对应的移动端兼容 JSON (`{"carStatus":"stop"}`) 发往执行层。

---

## 三、 通信保障机制

1.  **节流控制**：小程序摇杆上报间隔不低于 50ms。
2.  **强制重置**：Hi3861 在 50ms 超时未收到预期字符时，强制清空 UART 接收缓存。
3.  **模式权重**：手动控制指令（转向/摇杆）具有最高优先级，执行层在收到手动指令后将立即强行退出巡线或路径规划模式。
4.  **互锁逻辑**：执行层在收到小程序切换模式指令后的 500ms 内，将忽略来自传感器板物理按键产生的冗余模式切换请求。
