# ROS 激光雷达 SLAM 启动指南

> 使用 **Cartographer 2D** SLAM 后端。
> **IMU 为可选**，只连接雷达时脚本会自动以纯雷达模式启动。

## 硬件信息

| 设备 | 串口芯片 | 实际设备 | 波特率 |
|------|----------|----------|--------|
| 激光雷达 (LD-14P) | CH343 | `/dev/ttyCH343USB1` → `/dev/wheeltec_lidar` | 230400 |
| IMU (WHEELTEC H30) | CH343 | `/dev/ttyCH343USB0` → `/dev/yesense_imu` | 460800 |

> 注意：两个 CH343 USB 串口的编号可能因插拔顺序而交换，若启动后无数据，可交换 `/dev/ttyCH343USB0` 和 `/dev/ttyCH343USB1` 的映射。

---

## 一键启动（推荐）

```bash
cd /home/tzb/lidar
sudo ./start_slam.sh
```

该脚本自动完成：
1. 动态检测 CH343 串口并识别 Lidar/IMU 身份
2. 如果只连接了雷达，以**纯雷达模式**启动；如果检测到 IMU，自动启用 IMU 融合
3. 启动 Docker 容器（`osrf/ros:noetic-desktop`），只挂载实际存在的设备
4. 安装依赖（`serial`、`rviz-imu-plugin`、`cartographer_ros` 等）
5. 编译工作空间
6. 后台启动所有 ROS 节点

## 启动 RViz 可视化

```bash
xhost +local:root
sudo docker exec -it ros_container bash
source /catkin_ws/devel/setup.bash
rviz -d /catkin_ws/src/cartographer_ld06/rviz/cartographer.rviz
```

**RViz 显示内容**：
- **绿色点云**（LaserScan `/scan`）
- **灰色栅格地图**（Map `/map`）
- **轨迹**（`/trajectory_node_list`）
- **黄色立方体**（IMU `/imu`，姿态 orientation，仅在连接 IMU 时显示）
- **TF 坐标树**（`map → [odom] → base_link → laser/[imu_link]`）

## 常用诊断命令

### 查看节点列表
```bash
sudo docker exec ros_container bash -c "source /catkin_ws/devel/setup.bash && rosnode list"
```

### 查看话题列表
```bash
sudo docker exec ros_container bash -c "source /catkin_ws/devel/setup.bash && rostopic list"
```

### 查看激光雷达数据频率
```bash
sudo docker exec ros_container bash -c "source /catkin_ws/devel/setup.bash && rostopic hz /scan"
```

### 查看 IMU 数据频率
```bash
sudo docker exec ros_container bash -c "source /catkin_ws/devel/setup.bash && rostopic hz /imu"
```

### 查看位姿
```bash
sudo docker exec ros_container bash -c "source /catkin_ws/devel/setup.bash && rosrun tf tf_echo map base_link"
```

### 查看 TF 树（生成 PDF）
```bash
sudo docker exec ros_container bash -c "source /catkin_ws/devel/setup.bash && rosrun tf2_tools view_frames.py"
```

## 手动排查：交换串口映射

若启动后 `/scan` 或 `/imu` 无数据，可能是 USB 设备号交换了。

**在主机执行**：
```bash
# 停止当前容器
sudo docker stop ros_container && sudo docker rm ros_container

# 交换符号链接（将 USB0/USB1 互换）
sudo ln -sf /dev/ttyCH343USB0 /dev/wheeltec_lidar
sudo ln -sf /dev/ttyCH343USB1 /dev/yesense_imu
sudo chmod 666 /dev/ttyCH343USB0 /dev/ttyCH343USB1

# 重新启动
sudo ./start_slam.sh
```

**验证哪个设备对应哪个硬件**：
```bash
# 测试 IMU（应能看到 59 53 帧头）
stty -F /dev/ttyCH343USB0 460800 raw
sudo docker exec ros_container bash -c "timeout 2 head -c 100 /dev/ttyCH343USB0 | od -A x -t x1z | grep '59 53'"

# 测试激光雷达（应能看到 54 帧头）
stty -F /dev/ttyCH343USB1 230400 raw
sudo docker exec ros_container bash -c "timeout 2 head -c 100 /dev/ttyCH343USB1 | od -A x -t x1z | grep '54'"
```

## 注意事项

1. **地图更新**：Cartographer 每帧 scan 都会更新子图，全局优化每 25 个节点执行一次。
2. **IMU 显示的是姿态**：RViz 中的黄色立方体反映的是**旋转**（俯仰/横滚/偏航），不是位置。平移 IMU 模块不会改变它的显示，需要**旋转**模块才能看到变化。
3. **激光雷达视野**：不要放在桌面上（桌面会遮挡下方 180° 视野），建议手持或架高使用。
4. **首次启动较慢**：`start_slam.sh` 首次运行时需要安装 apt 依赖和编译，可能需要 3-5 分钟。
