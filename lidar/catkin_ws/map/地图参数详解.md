### 2. `resolution` — 分辨率

```yaml
resolution: 0.050000
```

| 说明 | 内容 |
|------|------|
| **含义** | 每个像素对应的实际物理尺寸（单位：米/像素） |
| **你的值** | **1 像素 = 5 厘米** |

**常见分辨率对比：**

| 分辨率 | 1 像素对应 | 适用场景 |
|--------|-----------|---------|
| `0.025` | 2.5 厘米 | 高精度室内、狭小空间 |
| `0.05`  | **5 厘米** | **中等精度，通用场景（你的配置）** |
| `0.10`  | 10 厘米 | 大范围室外、粗糙地图 |

**影响：**
- 分辨率越小 → 地图越精细 → 文件越大、计算量越大
- 分辨率越大 → 地图越粗糙 → 可能漏掉细小障碍物

### 3. `origin` — 地图原点

```yaml
origin: [-2.110601, -1.645580, 0.000000]
```

| 分量 | 值 | 含义 |
|------|-----|------|
| `origin[0]` | `-2.110601` | 地图左下角像素的 **X 坐标**（米） |
| `origin[1]` | `-1.645580` | 地图左下角像素的 **Y 坐标**（米） |
| `origin[2]` | `0.000000` | 地图的**偏航角**（弧度），`0` = 不旋转 |

**核心概念：**

`origin` 定义了地图**左下角像素 `[0, 0]`** 在**世界坐标系**中的位置。

```
            世界坐标系 (ROS /map frame)
                  Y ↑
                    |
   (-2.11, 4.45)    +------------------ (3.74, 4.45)
                    |                  |
                    |    ROS 原点      |
                    |       ● (0,0)    |
                    |   (42.2, 32.9)px |
   (-2.11, -1.65)   +------------------ (3.74, -1.65)
                    +----------------→ X

         ↑ 左下角像素 [0,0] 在世界坐标中为 (-2.11, -1.65)
```

#### 坐标转换公式

**世界坐标（米）→ 像素坐标：**

```
px = (world_x - origin_x) / resolution
py = (world_y - origin_y) / resolution
```

**像素坐标 → 世界坐标（米）：**

```
world_x = origin_x + px * resolution
world_y = origin_y + py * resolution
```

#### 你的地图具体计算

```
地图像素尺寸: 117 x 122
分辨率: 0.05 m/px

实际物理尺寸:
    宽度  = 117 × 0.05 = 5.85 米
    高度  = 122 × 0.05 = 6.10 米

地图覆盖区域:
    X: [-2.111, -2.111 + 5.85] = [-2.111, 3.739] 米
    Y: [-1.646, -1.646 + 6.10] = [-1.646, 4.454] 米

ROS 原点 (0, 0) 在地图像素中的位置:
    px = (0 - (-2.110601)) / 0.05 = 42.2
    py = (0 - (-1.645580)) / 0.05 = 32.9
```

> 你的 ROS 原点位于地图**内部**，像素坐标约为 `(42, 33)`。

### 4. `negate` — 颜色反转

```yaml
negate: 0
```

| 值 | 行为 | 含义 |
|-----|------|------|
| `0` | 不反转 | **0（黑）= 障碍物**，**255（白）= 空闲** |
| `1` | 反转 | 255 变 0，0 变 255（黑=空闲，白=障碍物） |

你的地图 `negate: 0`，即标准解读：
- 像素值 `0`（纯黑）→ **障碍物**
- 像素值 `255`（纯白）→ **空闲**

> 绝大多数 ROS 地图都使用 `negate: 0`。

### 5. `occupied_thresh` — 障碍物阈值

```yaml
occupied_thresh: 0.65
```

| 说明 | 内容 |
|------|------|
| **含义** | 判断一个格子为**障碍物**的概率阈值 |
| **计算** | `像素值 / 255 > 0.65` → 障碍物 |
| **你的阈值** | `0.65 × 255 = 165.75`，即像素值 **< 165** 为障碍 |

---

### 6. `free_thresh` — 空闲阈值

```yaml
free_thresh: 0.196
```

| 说明 | 内容 |
|------|------|
| **含义** | 判断一个格子为**空闲**的概率阈值 |
| **计算** | `像素值 / 255 < 0.196` → 空闲 |
| **你的阈值** | `0.196 × 255 = 49.98 ≈ 50`，即像素值 **> 50** 为空闲 |

## 三、三状态判定逻辑

ROS 地图中每个格子有三种状态，由两个阈值共同决定：

```
像素值归一化: p = pixel_value / 255

    ┌─────────────────────────────────────────┐
    │  p > occupied_thresh (0.65)             │
    │  → 障碍物 (occupied)                    │
    ├─────────────────────────────────────────┤
    │  p < free_thresh (0.196)                │
    │  → 空闲 (free)                          │
    ├─────────────────────────────────────────┤
    │  free_thresh ≤ p ≤ occupied_thresh      │
    │  → 未知 (unknown)                       │
    └─────────────────────────────────────────┘
```

### 你的地图实际值分布

你的地图是**二值地图**（建图时 SLAM 已完成收敛），只有纯黑和纯白：

| 像素值 | 归一化概率 | 理论状态 | 实际占比 |
|--------|-----------|---------|---------|
| `0` | `0` | 障碍物 | **9.8%** (1401 像素) |
| `1 ~ 49` | `0 ~ 0.192` | 空闲 | 0% |
| `50 ~ 165` | `0.196 ~ 0.647` | 未知 | 0% |
| `166 ~ 254` | `0.651 ~ 0.996` | 障碍物 | 0% |
| `255` | `1.0` | 空闲 | **90.2%** (12873 像素) |

> 虽然理论上 166~254 会被判为障碍，但你的地图实际只有 0 和 255 两种值，因此实际效果等价于：
> - `0` = 障碍物
> - `255` = 空闲

## 五、PGM 文件格式速览

```
P5                              ← 格式标识（P5 = 二进制灰度）
# CREATOR: map_saver.cpp ...    ← 注释行
117 122                         ← 宽度 高度（像素）
255                             ← 最大灰度值
[二进制像素数据...]              ← 117×122 = 14274 个字节
```

| 字段 | 值 | 说明 |
|------|-----|------|
| 格式 | `P5` | 二进制便携式灰度图 (Portable GrayMap Binary) |
| 尺寸 | `117 x 122` | 宽度 117 像素，高度 122 像素 |
| 最大灰度 | `255` | 单字节表示，范围 0~255 |
