# 基于OpenHarmony的智能小车物联网控制系统项目文档

## 文档说明

本文档为智能小车物联网控制系统的完整项目文档，涵盖项目概述、需求分析、系统设计、数据库设计、通信协议、功能模块设计、测试案例及测试报告等内容。所有子系统功能描述以实际代码实现为准。

## 第一部分 项目概述

### 1.1 项目背景

随着物联网技术的快速发展和智慧农业、智慧物流等领域的应用需求不断增长，智能小车作为移动物联网平台的重要载体，集成了环境感知、运动控制、远程通信等多项功能，成为嵌入式开发和物联网教学的典型案例。本项目基于华为OpenHarmony操作系统和Hi3861芯片，设计并实现了一套功能完备的智能小车物联网控制系统。

系统以设施农业大棚为典型应用场景，通过部署多源传感器网络采集土壤、环境参数，结合小车移动平台的灵活性，实现大棚环境巡回监测、智能灌溉指导、农业数据采集等功能。小车平台可搭载不同传感器模块，扩展至环境监测、智能搬运、病虫害巡检等多种应用场景。

### 1.2 项目目标

本项目以Hi3861开发板为核心控制单元，设计一套智能小车物联网控制系统。系统实现以下核心功能：小车运动控制（前进、后退、转向、速度调节）、自动模式控制（避障、巡线、路径规划）、环境数据采集（温度、湿度、光照）、智能照明控制、智能灌溉指导、以及远程监控与数据管理。系统同时提供微信小程序和HAP应用两种客户端，实现对小车的远程控制和数据查看。

### 1.3 系统特色

本系统的特色与创新点体现在以下几个方面：第一，系统采用华为自主研发的OpenHarmony物联网操作系统作为核心软件基座，基于Hi3861芯片实现，为开发者提供完整的嵌入式开发参考；第二，系统采用四层通信架构，从小程序/HAP应用到STM32执行层，协议清晰、层次分明；第三，系统集成了农业环境监测功能，可用于智慧农业场景下的移动巡检和数据采集；第四，系统提供微信小程序和HAP应用双客户端覆盖，HAP应用基于HarmonyOS 6.0.0开发，与OpenHarmony系统实现原生配合。

### 3.2 自动控制功能

#### 3.2.1 超声波避障

超声波避障功能通过发射超声波并接收回波来检测前方障碍物的距离。当检测到障碍物距离小于阈值（默认300mm）时，小车自动后退并转向，绕过障碍物后继续前进。

| 参数 | 默认值 | 说明 |
|-----|-------|------|
| 避障距离阈值 | 300mm | 触发避障的距离 |
| 后退时间 | 30ms | 后退持续时间 |
| 转向时间 | 30ms | 转向持续时间 |
| 手动优先时间 | 800ms | 手动控制打断自动的窗口 |

#### 3.2.2 红外巡线

红外巡线功能通过检测地面黑线的反射信号实现沿轨迹行驶。传感器板提供7通道ADC，可检测不同位置的黑线偏差，电机板根据偏差计算转向修正量，实现平滑的巡线行驶。

### 3.5 智能灌溉功能

智能灌溉功能根据环境参数计算推荐灌溉方案，指导农业生产。

#### 3.5.1 灌溉计算

系统根据温度、湿度、光照等参数，计算推荐灌溉水量：

```
water_amount = base_amount × temp_factor × humi_factor × light_factor × history_factor × season_factor
```

| 因子 | 基准值 | 调整规则 |
|-----|-------|---------|
| temp_factor | 25℃ | 每偏离1°C增加2% |
| humi_factor | 60% | 每低于1%增加1.5% |
| light_factor | 10000lux | 每增加100%增加10% |
| history_factor | 基准 | 1小时内灌溉则减少50% |
| season_factor | 春季基准 | 春1.1/夏1.3/秋1.0/冬0.7 |

## 第五部分 功能模块设计

### 5.1 PID控制算法

电机板采用增量式PID算法进行速度闭环控制：

```c
struct PID {
    float Kp;     // 0.5
    float Ki;     // 0.15
    float Kd;     // 0.01
    float target; // 目标速度
    float actual; // 实际速度
};
```

**速度档位映射**

| 档位 | PWM | 速度(mm/s) |
|-----|-----|-----------|
| 低 | 390 | 500 |
| 中 | 420 | 800 |
| 高 | 450 | 1100 |

### 5.2 超声波避障算法

```c
void auto_avoid_task(void) {
    if (auto_abstacle_flag && car_status != CAR_STATUS_OFF) {
        // 手动优先窗口检查
        if ((now - g_lastManualTick) > MANUAL_OVERRIDE_MS) {
            if (distance <= MIN_DISTANCE_VOL) {  // 300mm
                // 后退
                uart_send_control_cmd(CAR_STATUS_BACK);
                osDelay(30);
                // 转向(左右交替)
                uart_send_control_cmd(g_turnToggle ? LEFT : RIGHT);
                g_turnToggle ^= 1;
            } else {
                uart_send_control_cmd(CAR_STATUS_RUN);
            }
        }
    }
}
```

### 5.3 智能照明算法

**光照等级划分**

| 等级 | Lux范围 | 基础亮度 |
|-----|--------|---------|
| 暗 | <50 | 100% |
| 昏暗 | 50-200 | 75% |
| 低 | 200-500 | 50% |
| 正常 | 500-1000 | 30% |
| 明亮 | 1000-2000 | 15% |
| 强光 | >2000 | 0% |

**亮度计算**: 最终亮度 = 基础亮度 × 时间因子

### 5.4 智能灌溉算法

```python
def calculate_water_amount(self, temperature, humidity, lux, device_id):
    base = self.config.base_water_amount  # 500ml

    # 各因子计算
    temp_factor = 1.0 + (temperature - 25) * 0.02
    humi_factor = 1.0 + (60 - humidity) * 0.015
    light_factor = 1.0 + (lux / 10000) * 0.1

    # 历史因子检查
    recent = self.get_recent_irrigation(device_id)
    history_factor = 0.5 if recent else 1.0

    # 季节因子
    season = self.get_current_season()
    season_factor = {'spring': 1.1, 'summer': 1.3, 'autumn': 1.0, 'winter': 0.7}[season]

    water_amount = base * temp_factor * humi_factor * light_factor * history_factor * season_factor
    return max(100, min(2000, water_amount))  # 限制在100-2000ml
```
