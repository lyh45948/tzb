# OpenMV摄像头数据格式说明

## 概述

OpenMV摄像头通过USB虚拟串口(VCP)与PC通信，支持二进制图像数据传输和文本命令响应。

---

## 一、二进制图像数据格式

### 数据结构

```
┌─────────────────┬──────────────────────┬─────────────┐
│   数据长度       │     JPEG图像数据      │   校验和     │
│   (4字节)        │   (变长)              │   (1字节)    │
└─────────────────┴──────────────────────┴─────────────┘
```

### 字段说明

| 字段 | 长度 | 格式 | 说明 |
|------|------|------|------|
| 数据长度 | 4字节 | 小端无符号整数 (`<I`) | JPEG图像数据的字节数 |
| JPEG数据 | 变长 | 二进制 | 以`FF D8`开头，`FF D9`结尾 |
| 校验和 | 1字节 | 无符号字节 | JPEG数据所有字节之和的低8位 |

### 十六进制示例

```
A0 01 00 00  | FF D8 FF E0 ... FF D9 | 5A
└─ 长度=416 ─┘ └───── JPEG数据 ──────┘ └校验和┘
```

### 图像参数

| 参数 | 值 |
|------|-----|
| 分辨率 | QVGA (320×240) |
| 像素格式 | RGB565 |
| JPEG质量 | 70 (默认，可调整) |
| 典型JPEG大小 | 5-20KB |

## 二、文本命令格式

### PC → OpenMV 命令

| 命令 | 格式 | 说明 |
|------|------|------|
| 拍照 | `capture\n` | 捕获一帧图像并发送 |
| 视频流 | `stream\n` | 开始/停止视频流（切换） |
| 设置质量 | `quality:X\n` | X为1-100的整数 |
| 心跳检测 | `ping\n` | 测试连接 |
| 获取信息 | `info\n` | 获取设备信息 |

### OpenMV → PC 响应

| 命令 | 响应格式 | 示例 |
|------|----------|------|
| `capture` | 二进制图像数据 | [二进制格式] |
| `ping` | `PONG\n` | `50 4F 4E 47 0A` |
| `info` | `RES:320x240\nQUALITY:70\n` | 文本字符串 |
| `stream` | 无响应 | - |
| `quality:X` | 无响应 | - |

## 三、PC端接收逻辑

### 数据识别流程

```
1. 清空串口缓冲区
2. 发送 "capture" 命令
3. 等待300ms
4. 读取所有可用数据
5. 检测JPEG起始标记 (FF D8)
6. 查找JPEG结束标记 (FF D9)
7. 提取并解码JPEG数据
```

### 关键代码

```python
# 查找JPEG起始标记
jpeg_start = -1
for i in range(len(all_data) - 1):
    if all_data[i] == 0xFF and all_data[i+1] == 0xD8:
        jpeg_start = i
        break

# 查找JPEG结束标记
jpeg_end = -1
for i in range(jpeg_start + 2, len(all_data) - 1):
    if all_data[i] == 0xFF and all_data[i+1] == 0xD9:
        jpeg_end = i + 2
        break
```

## 四、错误处理

### 常见情况

| 情况 | 表现 | 处理方式 |
|------|------|----------|
| 未找到JPEG标记 | `jpeg_start < 0` | 可能是文本响应，检查内容 |
| JPEG数据不完整 | `jpeg_end < 0` | 丢弃数据，重新请求 |
| 图像过大 | `size > 500KB` | 降低JPEG质量或分辨率 |
| 超时 | 3秒内无数据 | 检查连接和脚本运行状态 |

### 文本响应识别

当收到非JPEG数据时，可能是以下文本响应：

```
- "capture" → OpenMV回显了命令，说明脚本未正确运行
- "PONG" → ping命令的正常响应
- "RES:320x240" → info命令的正常响应
```

## 五、视频流模式

当发送`stream`命令后，OpenMV进入视频流模式：

- 自动连续发送图像帧
- 帧率：约20fps (每帧间隔50ms)
- 再次发送`stream`命令停止

## 七、部署说明

### OpenMV端

1. 在OpenMV IDE中打开`openmv_camera.py`
2. 工具 → 将脚本保存到OpenMV
3. 工具 → 将当前脚本设置为main.py
4. 重启OpenMV设备

### PC端

```bash
# 启动GUI界面
python main_gui.py
```
